Component Map — What Was Built & Where

🤖 Copilot Studio Bot
  • Portal: make.powervirtualagents.microsoft.com
  • Owns: Topics, entities, variables, generative answers config
  • Publish button: pushes to registered channels
  • Dev/Test mode: runs independently of publish credentials
  • Expiry: None — but requires active license to publish
☁️ Azure Bot Service
  • Portal: portal.azure.com → Bot Services
  • Owns: Messaging endpoint, channel registrations
  • Key field: Microsoft App ID + App Password (client secret)
  • Expiry: App Password expires per App Registration secret policy
  • Must match: Secret in App Registration must match here
🔑 Azure AD App Registration
  • Portal: portal.azure.com → Microsoft Entra ID → App Registrations
  • Owns: App ID (permanent), Client Secret (expires)
  • Secret expiry: 1 or 2 years — most common failure point
  • Permissions: Bot Framework API, Teams, SharePoint (if direct)
  • Owner: Whoever created the registration (check Owners tab)
💬 Teams Channel
  • Registered in: Azure Bot Service → Channels → Microsoft Teams
  • Deployed via: Teams Admin Center → Manage Apps (org-wide) or user sideload
  • Owns: Teams app manifest, app ID within Teams
  • Expiry: Channel itself doesn't expire — breaks when Bot Service credentials go stale
  • Fix: Disconnect + reconnect channel after secret renewal
📂 SharePoint Knowledge Sources
  • Configured in: Copilot Studio → Knowledge → Add knowledge
  • Owns: Site URLs indexed for generative answers
  • Auth: Uses the bot's service identity / tenant permissions
  • Breaks when: SharePoint site permissions change, site deleted, or DLP policy blocks connector
  • Check: Knowledge tab shows green ✓ or red ✗ per source
⚡ Power Automate Flows
  • Configured in: Copilot Studio → Topics → Call an action
  • Auth: Connection credentials tied to the flow creator's account
  • Breaks when: Creator's password changes, account disabled, or MFA token expires
  • Check: power.microsoft.com → My Flows → look for Suspended status
  • Fix: Open flow → Edit connections → re-authenticate

Renew the Azure AD Client Secret (Teams Channel Fix)

ℹ This is the fix for: bot works in dev/test but Teams channel says "agent can't be reached" or stops responding after months of working.
#What to DoHow to Do It
1 Find the App Registration Go to portal.azure.com → Microsoft Entra ID → App Registrations → All Applications. Search for your bot name or "Copilot Studio." Open the registration.
2 Check the expired secret In the app registration, click Certificates & secrets → Client secrets tab. Look for a secret with a red or past Expires date. Note the Description name for reference.
3 Create a new secret Click + New client secret. Set Description (e.g. "Teams Bot 2026") and Expires to 24 months. Click Add. Immediately copy the Value — it is only shown once.
4 Update Azure Bot Service Go to portal.azure.com → Bot Services → your bot → Configuration. Paste the new secret into Microsoft App Password. Click Apply / Save.
5 Reconnect the Teams channel In Azure Bot Service → Channels → Microsoft Teams → click the channel → Delete to disconnect it. Then re-add it: + Add channel → Microsoft Teams → configure and save.
6 Re-publish from Copilot Studio Go to make.powervirtualagents.microsoft.com → your bot → Publish → Publish. Wait for confirmation. Test in Teams — allow 5–10 minutes for propagation.
7 Delete the old expired secret Back in Entra ID → App Registration → Certificates & secrets — delete the old expired secret to keep the list clean.
8 Set a renewal reminder Add a calendar reminder 30 days before the new secret's expiry date (24 months from today). Label it "Renew Teams Bot Azure Secret."

Fix Broken SharePoint Knowledge Sources

#What to DoHow to Do It
1 Check knowledge source status Copilot Studio → your bot → Knowledge tab. Each source shows a status icon. Red ✗ or "Failed" means the source is broken.
2 Verify SharePoint permissions Open the SharePoint site URL. Confirm the bot's service account or the Copilot connector still has at least Read access. Check Site Settings → Site Permissions.
3 Remove and re-add the source In Copilot Studio → Knowledge → click the broken source → Delete. Then + Add knowledge → SharePoint → paste the site URL again → Add. Allow up to 24 hours for re-indexing.
4 Check DLP policies Power Platform Admin Center → Policies → Data policies. Confirm SharePoint connector is not blocked in the environment the bot lives in. If it was moved to a restricted group, move it back to Business.
5 Re-publish after fix Copilot Studio → Publish after knowledge sources show green ✓. Test with a question that should pull from the SharePoint content.

Fix Suspended Power Automate Flows

#What to DoHow to Do It
1 Find suspended flows Go to power.microsoft.com → My Flows (or All Flows if admin). Look for flows with a Suspended or Warning badge — these are the ones the bot topics call.
2 Re-authenticate the connection Open the suspended flow → Edit → click on the step showing the broken connection (red border) → sign in again with valid credentials. Save the flow.
3 Change connection owner (if needed) If the original owner's account is gone, open the flow → Connections → + New connection → sign in with a service account or shared mailbox that won't be affected by staff turnover.
4 Test the flow In the flow editor, click Test → Manually → Run flow. Confirm it runs without errors before testing in the bot.

Troubleshoot: Can Publish to Demo Site but Not Teams

#CheckAction
1 Teams channel registration state Azure Bot Service → Channels. If the Microsoft Teams channel shows an error icon or "Disconnected," delete and re-add the channel (see Renew steps 5–6 above).
2 Azure AD client secret expired Most common cause. Follow the full Renew the Azure AD Client Secret procedure above.
3 Teams Admin Center app policy Teams Admin Center → Teams Apps → Setup Policies → check if the bot's app is blocked or if the policy changed. Ask your Teams admin to check org-wide app settings.
4 Copilot Studio license Power Platform Admin Center → Billing → Licenses → Copilot Studio. If the license shows expired or 0 available, the Publish button will succeed locally but fail to activate the Teams channel.
5 Bot registered in wrong environment Confirm the bot is published in the same Power Platform environment where the Teams app is registered. Environment mismatch causes silent publish failures.
