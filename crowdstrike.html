🛡️ CrowdStrike Falcon

Security

Manage endpoint protection, respond to detections, run Real Time Response sessions, and maintain sensor health across your fleet.

Falcon Console — Device Management

#What to DoHow to Do It
1Access the consoleGo to falcon.crowdstrike.com and sign in with SSO or your Falcon credentials.
2Find a deviceGo to Endpoint Security → Endpoint List. Search by hostname, IP, or username. Click the device to see details.
3Check sensor statusDevice detail shows Sensor Version, last check-in time, and prevention policy. A red or grey icon means the sensor is offline or outdated.
4Update sensor policyGo to Endpoint Security → Sensor Update Policies. Assign devices to the latest sensor version policy.
5Contain a host (network isolation)Device detail → Actions → Network Contain. The device is isolated from the network but still reachable via Falcon RTR. Use for active threats.
6Lift containmentDevice detail → Actions → Lift Network Containment. Only lift after confirming the threat is remediated.

Respond to Detections & Alerts

#What to DoHow to Do It
1View active detectionsGo to Endpoint Security → Detections. Filter by severity — Critical and High first. Click a detection to open the detail view.
2Review the process treeIn the detection detail, click View Process Tree. This shows the full execution chain — identify the root cause process.
3Quarantine a fileIn the detection detail → Actions → Quarantine File. This removes the file from the endpoint and stores it in Falcon for analysis.
4Block a file hashGo to Endpoint Security → IOC Management → Add IOC. Enter the SHA256 hash and set action to Block.
5Assign and close a detectionAssign the detection to yourself. After remediation, set status to Closed and add a comment documenting the action taken.
6Escalate if neededFor Critical severity or active ransomware indicators, escalate immediately. Contain the host first, then notify your security lead.

Real Time Response (RTR)

#What to DoHow to Do It
1Start an RTR sessionDevice detail → Actions → Real Time Response. Requires RTR permission in your Falcon role.
2List running processesIn the RTR console type: ps
3Kill a processkill <PID> — replace <PID> with the process ID from ps output.
4List filesls C:\Users\username\Downloads — browse the filesystem remotely.
5Get a fileget C:\path\to\suspicious.exe — retrieves the file to Falcon for analysis.
6Run a scriptUpload a PowerShell script to Custom Scripts first, then run it via runscript -CloudFile="scriptname.ps1"
7Check network connectionsnetstat — shows active connections. Look for unexpected outbound connections on unusual ports.

TaskCommand (PowerShell — Run as Admin)

Verify sensor is runningGet-Process -Name CSFalconService -ErrorAction SilentlyContinue | Select-Object Name, Id, CPU
