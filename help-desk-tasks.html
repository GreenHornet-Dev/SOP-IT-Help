🎟️ Common Help Desk Tasks

Help Desk

Frequently used IT support tasks — password resets, account unlocks, remote assistance, and performance fixes.

Passwords & Account Unlocks

#What to DoHow to Do It
1Password reset (self-service)Direct user to Okta or the company login portal > click Forgot Password. Follow reset steps via email.
2Unlock account (Okta)In the Okta Admin Console go to Directory > People, find the user, click Unlock.
3Unlock account (Active Directory)In ADUC or via PowerShell: Unlock-ADAccount -Identity "username". Confirm with the user.
4Reset AD password (admin)Use the Quick Reference PowerShell below, or right-click the user in ADUC > Reset Password.

Remote Desktop Assistance

#What to DoHow to Do It
1Connect to user's machineUse LogMeIn Resolve or the assigned remote tool. Always get the user's verbal permission before connecting.
2RDP to a machinePress Win+R > type mstsc > enter the computer name or IP. Sign in with admin credentials.

Hardware & Software Requests

#What to DoHow to Do It
1New hardware requestSubmit a ticket in the help desk system with the user's name, department, and hardware needed.
2Software installationCheck if the software is approved. Deploy via Lansweeper or RMM tool, or guide the user through manual install.

Slow Computer Fix

#What to DoHow to Do It
1Check CPU & RAM usageOpen Task Manager (Ctrl+Shift+Esc). Look for processes using high CPU or memory.
2Restart and clear temp filesRestart the computer, then run cleanmgr or use the one-liner below to clear temp files.
3Run Windows UpdateInstall any pending updates — outdated systems often run slowly due to background update activity.

Quick Reference — One-Liners

TaskCommand (PowerShell — Run as Admin)
Unlock AD accountUnlock-ADAccount -Identity "username"
Reset AD passwordSet-ADAccountPassword -Identity "user" -NewPassword (ConvertTo-SecureString "TempP@ss1!" -AsPlainText -Force) -Reset
Find locked-out accountsSearch-ADAccount -LockedOut | Select-Object Name, SamAccountName, LockedOut
Check account statusGet-ADUser "username" -Properties LockedOut, Enabled, LastLogonDate | Select-Object Name, Enabled, LockedOut, LastLogonDate
Clear user temp filesRemove-Item -Path $env:TEMP\* -Recurse -Force -ErrorAction SilentlyContinue
Check top CPU processesGet-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, CPU, WorkingSet
Open RDPStart-Process mstsc
