2. API Calls from VS Code — REST Client & Python

Method 1 — REST Client Extension (.http files)

Create a file called test-api.http. Click Send Request above any request block. Response appears in a split pane — no browser, no Postman.

### Lansweeper GraphQL
POST https://api.lansweeper.com/api/v2/graphql
Content-Type: application/json
Authorization: Bearer {{LS_PAT}}

{
  "query": "{ Site { site { assetBasicInfo(where: { assetBasicInfo: { name: { like: \"%PC-001%\" } } }) { name ipAddress userName } } } }"
}

###

### Jira — My Open Tickets
GET https://{{JIRA_HOST}}/rest/api/3/search?jql=assignee%3DcurrentUser()%20AND%20resolution%3DUnresolved
Authorization: Basic {{JIRA_AUTH}}
Accept: application/json

###

### Power Automate Webhook — Trigger a Flow
POST {{PA_WEBHOOK_URL}}
Content-Type: application/json

{
  "trigger": "manual",
  "source": "vscode-test"

Create a matching .env file (or use the REST Client's environment file .vscode/settings.json):

Reading API Responses — What to Look For

Response partWhat it tells you
Status 200Success — check the body for your data
Status 401Auth failed — wrong token, wrong format, or expired
Status 403Token valid but no permission — check scopes/roles
Status 404Wrong URL — check endpoint path and host
Status 429Rate limited — too many requests, add delay or cache results
CORS errorBrowser-only restriction — run from Python instead, or use a proxy

3. Indexing & Parsing — JSON, CSV, and Data Formats

APIs return JSON. Reports come as CSV or Excel. Parsing is how you extract just the fields you care about and get them into a usable format.

JSON — Navigate Nested Data

import json

# Load from API response
data = response.json()

# Navigate nested structure — Jira example
for issue in data["issues"]:
    key     = issue["key"]                          # "IT-123"
    summary = issue["fields"]["summary"]            # ticket title
    status  = issue["fields"]["status"]["name"]     # "In Progress"
    assignee = issue["fields"]["assignee"]["displayName"] if issue["fields"]["assignee"] else "Unassigned"
    print(f"{key} | {status} | {assignee} | {summary}")

# Lansweeper GraphQL example
assets = data["data"]["Site"]["site"]["assetBasicInfo"]
for a in assets:
    print(a["name"], a["ipAddress"], a["userName"], a["lastSeen"])

JSON — Format and Pretty-Print

import json

raw = response.json()

# Pretty-print to console
print(json.dumps(raw, indent=2))

# Save formatted JSON to file
with open("output.json", "w") as f:
    json.dump(raw, f, indent=2)

# VS Code: open any .json file → Shift+Alt+F to auto-format it

CSV — Read and Write

import csv

# Read a CSV
with open("assets.csv", newline="") as f:
    reader = csv.DictReader(f)  # uses first row as field names
    for row in reader:
        print(row["Name"], row["IPAddress"], row["Username"])

# Write a CSV from API data
with open("report.csv", "w", newline="") as f:
    writer = csv.DictWriter(f, fieldnames=["key","summary","status","assignee"])
    writer.writeheader()
    for issue in data["issues"]:
        writer.writerow({
            "key":      issue["key"],
            "summary":  issue["fields"]["summary"],
            "status":   issue["fields"]["status"]["name"],
            "assignee": issue["fields"]["assignee"]["displayName"] if issue["fields"]["assignee"] else ""
        })

Indexing — Build a Lookup Dictionary

When you need to cross-reference two data sources (e.g., Jira assignees vs Lansweeper users), index by a shared key:

# Index Lansweeper assets by username for fast lookup
asset_index = {}
for asset in assets:
    user = asset["userName"].lower() if asset["userName"] else "unknown"
    asset_index.setdefault(user, []).append(asset)

# Now cross-reference: for each Jira ticket, find the assignee's assets
for issue in jira_data["issues"]:
    assignee = issue["fields"]["assignee"]
    if assignee:
        username = assignee["emailAddress"].split("@")[0].lower()
        machines = asset_index.get(username, [])
        print(f"{issue['key']} — {username} — machines: {[m['name'] for m in machines]}")

4. Power Automate — Next Level Integration

The Architecture

PatternHow it works
VS Code → PA (push)Python script POSTs JSON to a PA HTTP trigger webhook → flow runs
PA → VS Code API (pull)PA HTTP action calls a Flask/FastAPI endpoint you host locally or on Azure
PA → SharePoint (write)PA creates/updates list items, uploads files, sets metadata/labels
PA → Teams/Email (notify)PA posts adaptive cards or emails based on data from your script

Pattern 1 — Python Triggers a Power Automate Flow

import requests
from dotenv import load_dotenv
import os

load_dotenv()
webhook = os.getenv("PA_WEBHOOK_URL")

payload = {
    "event": "asset_offline",
    "hostname": "PC-FINANCE-04",
    "last_seen": "2026-04-07",
    "assigned_to": "jsmith@newsbank.com",
    "department": "Finance"
}

r = requests.post(webhook, json=payload)
print(f"Flow triggered: {r.status_code}")

In Power Automate, the flow receives this JSON, parses the body, and can email the assigned user, create a Jira ticket, or update a SharePoint list — all automatically.

Pattern 2 — Build a Local API for Power Automate to Call

Use Flask to expose a local endpoint. PA's HTTP connector calls it on a schedule or trigger.

from flask import Flask, request, jsonify
import pyodbc

app = Flask(__name__)

@app.route("/api/assets/offline", methods=["GET"])
def offline_assets():
    conn = pyodbc.connect("DRIVER={SQL Server};SERVER=your-server;DATABASE=lansweeperdb;Trusted_Connection=yes")
    cursor = conn.cursor()
    cursor.execute("SELECT Name, IPAddress, Username FROM tblAssets WHERE LastActiveScan < DATEADD(day,-30,GETDATE())")
    rows = [{"name": r[0], "ip": r[1], "user": r[2]} for r in cursor.fetchall()]
    return jsonify({"count": len(rows), "assets": rows})

if __name__ == "__main__":
    app.run(port=5000)

In Power Automate: HTTP action → GET http://localhost:5000/api/assets/offline → parse JSON response → send Teams alert with the list.

For production: Deploy to Azure App Service or a Windows server running waitress as the WSGI server — same code, accessible from PA without localhost.

5. SharePoint Tag Redaction — HR & Finance

Sensitivity labels classify and optionally redact content in SharePoint — protecting HR (PII, personnel records) and Finance (payroll, contracts) data from unauthorized access while keeping it usable by the right people.

Three Layers of Protection

LayerWhat it does
Sensitivity Labels (Microsoft Purview)Tag documents/sites as Confidential, HR-Only, Finance-Only — enforces encryption, access, and watermarks
SharePoint Metadata TagsCustom columns (Department, Classification) for filtering and Power Automate targeting
Permissions / SharingSite-level or document library-level access — HR group only, Finance group only

Step 1 — Create Sensitivity Labels in Microsoft Purview

StepAction
1Go to purview.microsoft.com → Information Protection → Labels
2Click Create a label — name it (e.g., HR-Confidential, Finance-Restricted)
3Configure scope: Files & emails + Groups & sites
4Enable Encryption → assign permissions: HR group gets Co-Author, all others get none
5Enable Content marking → add watermark: "HR CONFIDENTIAL — DO NOT DISTRIBUTE"
6Publish the label via a Label Policy → assign to HR and Finance users

Step 2 — Apply Labels to SharePoint Sites & Libraries

StepAction
1In the HR SharePoint site: Settings (gear) → Site information → Sensitivity label
2Select HR-Confidential — this applies the label to the entire site
3For a specific library: Library Settings → Sensitivity labels → Apply default label
4All new documents uploaded automatically inherit the site/library label

Step 3 — Auto-Label Rules (Power Automate + Purview)

Automatically tag documents based on content — SSNs, salary data, account numbers:

StepAction
1In Purview → Auto-labeling policies → Create policy
2Choose sensitive info types: U.S. Social Security Number, Credit Card Number, ABA Routing Number
3Set location: your HR and Finance SharePoint sites
4Run in Simulation mode first — see what would be labeled before applying
5Enable the policy — documents with matching content are auto-labeled on upload or edit

Step 4 — Power Automate Flow to Flag Mislabeled Documents

# Python: call Microsoft Graph to list SharePoint files missing a label
import requests, msal, os
from dotenv import load_dotenv
load_dotenv()

app = msal.ConfidentialClientApplication(
    os.getenv("AZURE_CLIENT_ID"),
    authority=f"https://login.microsoftonline.com/{os.getenv('AZURE_TENANT_ID')}",
    client_credential=os.getenv("AZURE_CLIENT_SECRET")
)
token = app.acquire_token_for_client(["https://graph.microsoft.com/.default"])["access_token"]

# List files in HR SharePoint library
site_id  = os.getenv("SP_HR_SITE_ID")
list_id  = os.getenv("SP_HR_LIST_ID")
r = requests.get(
    f"https://graph.microsoft.com/v1.0/sites/{site_id}/lists/{list_id}/items?expand=fields",
    headers={"Authorization": f"Bearer {token}"}
)
for item in r.json().get("value", []):
    label = item["fields"].get("_ComplianceTag", "")
    name  = item["fields"].get("FileLeafRef", "")
    if not label:
        print(f"⚠️  No label: {name}")

Redaction vs Classification — What's What

TermWhat it means in practice
Classification / TaggingLabel applied — document is marked as sensitive. Access still possible by permitted users.
EncryptionFile is encrypted — only licensed, permitted users can open it regardless of where it's shared
RedactionSpecific content blacked out — requires Microsoft Purview eDiscovery or a third-party tool (not native in SharePoint labels)
DLP PolicyData Loss Prevention — blocks sharing labeled docs externally, alerts IT when attempted

DLP Policy — Block External Sharing of HR/Finance Docs

StepAction
1Purview → Data Loss Prevention → Policies → Create policy
2Select template: Financial Data or U.S. Personally Identifiable Information
3Set location: SharePoint sites (select HR and Finance sites)
4Rule: if label = HR-Confidential AND shared externally → Block + notify IT
5Enable User override with justification — so users can explain legitimate external shares
6Run in Test mode for 2 weeks before enforcing
