🖥️ LogMeIn Resolve

Remote Support

Start remote support sessions, deploy unattended access, manage devices, and troubleshoot common LogMeIn Resolve issues.

Start a Remote Support Session

#What to DoHow to Do It
1Sign in to ResolveGo to resolve.logmeininc.com and sign in with your IT credentials.
2Start a sessionClick Start Session. Enter the user's email or copy the support link and send it via chat or email.
3User joinsUser clicks the link, downloads the small agent, and clicks Allow to grant you access.
4Get verbal permissionAlways get the user's verbal or written permission before connecting. Sessions are logged.
5Use session toolsUse the toolbar to chat, transfer files, reboot remotely, or run commands during the session.
6End the sessionClick End Session in the toolbar. The agent uninstalls automatically (unless unattended access is deployed).

Unattended Access

#What to DoHow to Do It
1Deploy the agentIn Resolve console go to Computers → Add Computers. Download the installer or deploy via your RMM / GPO.
2Connect without user presentGo to Computers, find the device (shows green when online), and click Connect.
3Admin-level accessUnattended sessions run with the service-level permissions of the installed agent — no user elevation needed.
4Silent deploy via MSIDownload the MSI from the Resolve console. Deploy via GPO, Lansweeper, or run: msiexec /i ResolveAgent.msi /quiet

Troubleshooting

#IssueFix
1User can't join sessionCheck that the user's browser allows the download. Try sending the direct link again or use the PIN code option instead.
2Slow or laggy sessionReduce session quality in Settings → Quality. Check user's network — ask them to move closer to Wi-Fi or plug in Ethernet.
3Unattended device shows offlineCheck the Resolve agent service on the device: Get-Service -Name "LogMeIn". Restart if stopped: Restart-Service "LogMeIn" -Force
4Can't see desktop (black screen)Ask user to unlock their screen. If unattended, check if the machine is on the Windows login screen — use the Send Ctrl+Alt+Del button in the toolbar.
5File transfer not workingEnsure the Resolve agent is up to date. Try the File Manager tab instead of drag-and-drop.

Quick Reference — One-Liners

TaskCommand (PowerShell — Run as Admin)
Check LogMeIn agent serviceGet-Service -Name "LogMeIn" | Select-Object Name, Status, StartType
Restart LogMeIn agentRestart-Service "LogMeIn" -Force
Silent install agent (MSI)msiexec /i "ResolveAgent.msi" /quiet /norestart
Check agent versionGet-WmiObject -Class Win32_Product | Where-Object {$_.Name -like "*LogMeIn*"} | Select-Object Name, Version
Uninstall agentGet-WmiObject -Class Win32_Product | Where-Object {$_.Name -like "*LogMeIn*"} | ForEach-Object {$_.Uninstall()}
