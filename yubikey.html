🔑 YubiKey Reprogram SOP

Security MFA

1. When to Reprogram a YubiKey

ScenarioAction Required
New hire receiving a previously-used YubiKeyFull reset + re-enroll in Okta with new user account
Employee forgot their YubiKey PINReset FIDO2 PIN → re-enroll authenticator in Okta
YubiKey blocked / locked (too many wrong PINs)Full FIDO2 reset → re-enroll
Departing employee — wipe before re-issuingFull reset all applets (FIDO2, OTP, PIV)
User getting a replacement key (lost/damaged)Deactivate old key in Okta → enroll new key
OTP slot producing wrong codes or mis-mappedWipe OTP slots → reprogram Slot 1 / Slot 2
PIV smart card cert expired or wrong cert loadedPIV reset → re-provision cert from CA or Okta

Important: Resetting a YubiKey is irreversible. All stored credentials, PINs, and keys on the reset applet are permanently deleted. Confirm the user is removed from Okta authenticators before resetting, or they will be locked out.

2. Prerequisites & Tools

Install YubiKey Manager

YubiKey Manager is the official Yubico tool for managing and reprogramming YubiKeys. Install via winget (recommended) or download from yubico.com.

# Install YubiKey Manager GUI + CLI via winget
winget install Yubico.YubikeyManager

# Verify ykman CLI is available
ykman --version

The GUI (YubiKey Manager) is sufficient for most resets. The CLI (ykman) is faster for bulk operations or scripting.

3. Identify YubiKey Type & Current Config

Always identify the device before resetting. Different firmware versions have different reset procedures.

# Plug in YubiKey, then run:
ykman info

Example output:

Serial number: 12345678
Firmware version: 5.4.3
Form factor: Keychain (USB-A)
Enabled USB interfaces: OTP, FIDO, CCID

Check Which Slots / Applets Are In Use

# FIDO2 credentials count
ykman fido info

# OTP slot status

4. Reset the YubiKey

Before resetting: Remove the user's YubiKey authenticator from Okta Admin Console first (Security → Multifactor → Users → select user → remove YubiKey). This prevents lockout during the re-enrollment step.

Option A — Full Reset via YubiKey Manager GUI

StepAction
1Plug in the YubiKey
2Open YubiKey Manager
3Click Applications in the top nav
4For FIDO2: click FIDO2 → Reset FIDO2
5For OTP: click OTP → select slot → Delete
6For PIV: click PIV → Reset PIV
7Confirm each reset — the key may ask you to remove and re-insert it

Option B — Reset via CLI (ykman)

# Reset FIDO2 (WebAuthn / FIDO U2F credentials + PIN)
# Key must be re-inserted within 5 seconds when prompted
ykman fido reset

# Reset OTP slots (wipes Slot 1 and Slot 2)
ykman otp delete 1
ykman otp delete 2

# Reset PIV (wipes all certs and resets PIN/PUK to defaults: 123456 / 12345678)
ykman piv reset

# Reset OATH (TOTP/HOTP accounts)
ykman oath reset

# Reset OpenPGP
ykman openpgp reset

FIDO2 reset requires physical touch: After running ykman fido reset, the LED on the key will flash. Remove and re-insert the key within 5 seconds, then touch the gold contact when it blinks again to confirm the reset.

Full Wipe (All Applets)

# Wipe everything — use for departing employee key reuse
ykman fido reset
ykman otp delete 1
ykman otp delete 2
ykman piv reset
ykman oath reset
Write-Host "YubiKey fully wiped and ready for re-issue."

5. Reprogram OTP Slot (Optional)

Most Okta environments use FIDO2/WebAuthn for YubiKey MFA — skip this section unless your org uses Yubico OTP or HMAC-SHA1 challenge-response. If Okta is configured for FIDO2, jump to Section 6.

Program Slot 1 — Yubico OTP (Default)

Slot 1 is activated by a short touch (≤ 0.3 s). This is the standard Yubico OTP configuration used for basic TOTP-style validation.

# Program Slot 1 with Yubico OTP (generates new private key + public ID automatically)
ykman otp yubiotp 1 --generate

The command outputs the new Public Identity and Private Key — upload these to your Yubico Validation Server or YubiCloud if applicable.

Program Slot 2 — HMAC-SHA1 Challenge-Response

Slot 2 is activated by a long touch (≥ 2.5 s). Used for offline MFA or KeePassXC integration.

# Program Slot 2 with HMAC-SHA1 Challenge-Response (auto-generate key)
ykman otp chalresp 2 --generate --touch

The --touch flag requires physical touch to respond to challenges, adding a presence check.

Verify OTP Slots After Programming

ykman otp info

Expected output: Slot 1: programmed, Slot 2: programmed.

6. Re-enroll FIDO2 in Okta MFA

After the FIDO2 reset, the user's YubiKey must be re-enrolled as a new WebAuthn authenticator in Okta. There are two paths: user self-enrollment from Okta dashboard, or admin-initiated enrollment link.

Step 1 — Remove Old YubiKey Authenticator (Admin)

StepAction
1Log in to Okta Admin Console → Directory → People
2Search for and open the user's profile
3Click the Security tab
4Under Enrolled Authenticators, find the YubiKey / Security Key entry
5Click Remove next to the old authenticator
6Confirm removal

Step 2 — User Self-Enrollment (Preferred)

StepAction
1User logs in to Okta dashboard → clicks their name → Settings
2Scrolls to Security Methods (or Extra Verification)
3Clicks Set up next to Security Key or Biometric Authenticator (WebAuthn)
4Clicks Set up in the popup → browser prompts: "Use your security key"
5Inserts the YubiKey (if not already) and touches the gold contact
6If prompted to set a PIN, choose a 4–8 digit PIN and confirm it
7Okta confirms: "Your security key was enrolled successfully"

Step 3 — Admin-Initiated Enrollment Link (If User Can't Self-Enroll)

StepAction
1Admin Console → Directory → People → open user
2Click More Actions → Reset Multifactor
3Select Security Key or Biometric (WebAuthn) → click Reset
4User will be prompted to enroll a new authenticator on next login
5Walk user through the self-enrollment steps above at next login

If Okta Requires an Existing MFA to Enroll a New One

Some Okta policies require a currently-enrolled factor to add a new one. If the user has no other factor (no phone, no backup code):

StepAction
1Admin: temporarily reset all MFA factors for the user (More Actions → Reset Multifactor → Reset All)
2User logs in with password only → Okta prompts to set up MFA
3User enrolls YubiKey as the primary factor
4Admin re-enforces any policy requirement after enrollment

Admin Console → user Security tabNew WebAuthn authenticator appears with a recent enrollment date
Run ykman fido infoShows PIN is set + credential count is 1 (or more if other FIDO2 sites enrolled)
Try wrong PIN 3× (test)Key locks temporarily — run ykman fido access change-pin to reset after correct PIN retry

Check FIDO2 PIN Retry Count

# Shows how many PIN attempts remain before lockout
ykman fido info

If PIN retries remaining: 0 — the key is blocked. A full FIDO2 reset is required (Section 4). All FIDO2 credentials will be lost.

9. Troubleshooting

ProblemSolution
ykman info shows no device foundTry a different USB port. On Windows: Device Manager → check for unknown HID device. Reinstall YubiKey Manager.
FIDO2 reset says "touch required" but nothing happensRemove the key entirely after the command runs, then re-insert within 5 seconds and touch the gold contact once.
Okta enrollment: browser says "No available authenticator"Ensure the key is plugged into a USB port (not a hub). Try Chrome or Edge — Firefox requires FIDO2 permissions flag in some versions.
Okta enrollment fails with "This security key is already registered"The old FIDO2 credential was not removed from Okta. Admin: remove it under the user's Security tab, then retry enrollment.
FIDO2 PIN blocked (retries = 0)Full FIDO2 reset required: ykman fido reset. All stored FIDO2 credentials will be lost and must be re-enrolled.
After reset, Okta still asks for old YubiKeyAdmin must remove the old authenticator entry from the user's Okta profile before re-enrollment (Section 6 Step 1).
YubiKey touch not detected during Okta sign-inCheck USB connection. Try a direct USB port. On Windows 11: Settings → Bluetooth & Devices → verify key is listed as HID device.
PIV reset didn't clear smart card certAfter ykman piv reset, default PIN is 123456 and PUK is 12345678. Change them immediately: ykman piv access change-pin and ykman piv access change-puk.
