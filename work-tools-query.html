1. Overview & Redacted Build Pattern

The Work Tools Query Hub connects Clippy to your IT management stack: Jira (ticket tracking), Lansweeper (asset inventory), and Power Automate (flow automation). All credentials are stored in your browser's localStorage only — nothing sensitive is in this repository or published to GitHub Pages.

Redacted Build Architecture

LayerWhat it contains
Public repo (GitHub Pages)All HTML, CSS, JS — zero credentials, zero keys. Safe to publish.
Browser localStorageJira URL + token, Lansweeper PAT, Power Automate webhook URLs. Set at work, never leaves the device.
Private newsbank repoFuture: full config baked in, CORS proxy, SQL endpoint — for controlled-environment deployment.

2. Work Config Setup

Type work config in Clippy (the chat bubble, bottom-right) to open the config form. All values are saved to localStorage in your browser — they survive page refreshes but are cleared if you clear site data.

Jira Setup

StepAction
1Go to your Jira instance → Profile > Account Settings > Security > API Tokens
2Click Create API token — name it ClippyLocal
3Copy the token (you won't see it again)
4In Clippy → work config → paste your Jira URL (e.g. https://newsbank.atlassian.net)
5Paste your Atlassian email and the API token
6Click Save Jira & LS
7Test by typing my tickets in Clippy

Note: Jira REST API v3 requires Basic auth: base64(email:token). Clippy builds this for you — just provide email and token separately.

Lansweeper Setup

StepAction
1Log in to app.lansweeper.com
2Go to Settings (gear icon, bottom-left) > Developer Tools > API Clients
3Click Add New API Client → Personal Access Token (PAT)
4Name it ClippyLocal — copy the access token
5In Clippy → work config → paste the GraphQL endpoint: https://api.lansweeper.com/api/v2/graphql
6Paste the PAT — optionally enter your site name to filter results
7Test by typing find HOSTNAME in Clippy

Power Automate Webhook Setup

StepAction
1Open your flow in make.powerautomate.com
2Add a When an HTTP request is received trigger (if not already present)
3Save the flow — Power Automate generates an HTTPS POST URL
4Copy the generated URL
5In Clippy → work config → enter a name and paste the URL → click + Save Flow
6Type my flows to see and run saved flows

SQL Endpoint (Optional — for Live Query Execution)

By default, SQL presets are copy-only — paste into SSMS and run there. If you have a Lansweeper SQL proxy endpoint (a local API that accepts POST with {query} and returns JSON rows), add it in work config under SQL Endpoint URL. Without it, all SQL features still work — just copy to clipboard.

3. SQL Query Library — Lansweeper Database Queries

These T-SQL queries run against the Lansweeper on-premise database (lansweeperdb). In Clippy, type sql to open the browser and copy any query. If a SQL proxy endpoint is configured, use ▶ Run to execute live.

Clippy command: sql — opens full library. sql software, sql offline, etc. — filtered view.

4. GraphQL Runner — Live Lansweeper API Queries

The GraphQL runner executes live queries against the Lansweeper Data API. Requires your Lansweeper PAT configured in work config (or entered directly in the runner). Type graphql in Clippy to open the runner.

Endpoint: https://api.lansweeper.com/api/v2/graphql — all requests use HTTP POST with Authorization: Bearer <PAT>.

GraphQL Preset Reference

PresetDescriptionPlaceholder
Find Asset by NameReturns IP, user, last seen, OS for matching hostnamesHOSTNAME
Assets in DepartmentAll assets where custom department field matches"IT"
Find by Serial NumberExact serial match — returns asset summarySERIAL
Software on AssetAll software installed on a specific hostnameHOSTNAME
Assets for a UserAll assets where userName contains the search stringusername
Assets Not Seen 30+ DaysStale assets — requires ISO 8601 date for DATEOFFSETDATEOFFSET

Example — Find Asset by Name

{
  Site {
    site {
      assetBasicInfo(
        where: { assetBasicInfo: { name: { like: "%HOSTNAME%" } } }
        limit: 10
      ) {
        name
        ipAddress
        userName
        lastSeen
        operatingSystem
      }
    }
  }
}

How to Use the Runner

StepAction
1Type graphql in Clippy — the runner panel opens
2Click a preset button to load a query
3Edit the placeholder values (HOSTNAME, SERIAL, etc.)
4Verify the endpoint and token are filled (auto-populated from work config)
5Click ▶ Run Query — results render as pretty JSON
6Click 📋 Copy JSON to copy full results

Date Offset Format (for Stale Assets)

Lansweeper's lt (less than) date filter accepts ISO 8601 format. To query assets not seen in 30 days, calculate the cutoff date:

# PowerShell — get the date 30 days ago in ISO 8601 format
(Get-Date).AddDays(-30).ToString("yyyy-MM-ddTHH:mm:ssZ")

Paste the output (e.g. 2026-04-08T00:00:00Z) into the DATEOFFSET placeholder in the preset.

5. JQL Preset Browser — Jira Query Language

JQL (Jira Query Language) lets you filter and find tickets with precision. Type jql in Clippy to open the preset browser with copy buttons and direct "Open in Jira" deep-links (requires Jira URL configured in work config).

JQL Preset Reference

PresetJQL Query
My Open Tickets assignee = currentUser() AND resolution = Unresolved ORDER BY updated DESC
Current Sprint — IT Project project = IT AND sprint in openSprints() ORDER BY priority DESC
Critical / High Priority priority in (Critical, High) AND resolution = Unresolved ORDER BY created ASC
Updated Today updated >= startOfDay() ORDER BY updated DESC
Overdue duedate < now() AND resolution = Unresolved ORDER BY duedate ASC
Created by Me This Week reporter = currentUser() AND created >= startOfWeek() ORDER BY created DESC
Unassigned Open Tickets assignee is EMPTY AND resolution = Unresolved ORDER BY created ASC
Waiting on Me assignee = currentUser() AND status = "Waiting for Support" ORDER BY updated ASC

JQL Quick Reference — Common Operators

OperatorMeaning / Example
= / !=Exact match — status = "In Progress"
in / not inList match — priority in (High, Critical)
~Contains text — summary ~ "Teams outage"
is EMPTYField not set — assignee is EMPTY
>= / <=Date range — created >= "2026-05-01"
AND / ORCombine conditions — project = IT AND status != Done
ORDER BYSort — ORDER BY priority DESC, updated DESC
currentUser()Dynamic — resolves to whoever is logged in
startOfDay() / startOfWeek()Rolling time window — no hardcoded dates needed

Building a Custom Report in Lansweeper Portal

StepAction
1Go to Reports > Create New Report
2Choose asset type: Windows, All Assets, Network Device, etc.
3Add columns — custom fields appear under the Custom category
4Filter by OS, last seen, IP range, or department
5Save report — optionally Expose to API for Power Automate or Power BI
6Pin to dashboard as a Table widget for at-a-glance monitoring

Expose a Report to the GraphQL API

  1. Open the saved report in Lansweeper portal
  2. Click Report Settings > Expose to API
  3. Note the report ID from the URL
  4. Query it via GraphQL under authorizedReports using your PAT
  5. Use in a scheduled Power Automate flow to feed SharePoint or Power BI
